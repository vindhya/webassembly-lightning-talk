What is WebAssembly (Wasm)?

  • Standard to bring native code to web browsers
    • Supported in all major browsers
  • Compilation target for native programming languages such as, C, C++, Go, Rust
Compilation target to build a .wasm file for native programming languages like C, C++, C#, Go, and Rust

Compilation target? Huh?

  • JS is interpreted and compiled in the browser
  • Wasm is partially compiled ahead of time

JavaScript is an interpreted language which means that, at a very high level, web browsers go through each line of source code and parse, optimize, and execute the code at runtime.

When traditional languages are compiled they do some of this work ahead of when the code is actually executed - what is generated when these languages are compiled is called the Intermediate Representation. WebAssembly is a type of Intermediate Representation that web browsers understand.

Source: Lin Clark, Mozilla
The Intermediate Representation (IR) is the bytecode that the compilers generate from source code. Machines can then take that representation and execute it.
Source: Lin Clark, Mozilla

Benefits

  • Fast, more consistently performant
  • Smaller downloads
  • All the benefits of web apps: linkable, secure, composable

Because WebAssembly is already optimized for web browsers, its applications run more quickly and the file sizes are smaller.

In addition, it gets all the other benefits that web apps have

Examples

  • AutoCAD now works on the web
  • Windows 2000 emulation
  • Adobe Lightroom web app: A core image-editing library written in asm.js (pre-cursor to WebAssembly)
  • eBay Barcode Scanner for web (article)
